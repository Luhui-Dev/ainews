Moonshot AI / Kimi 2026-07-17

Moonshot 发布 Kimi K3 并开放 Kimi Work、Kimi Code 和 API

月之暗面正式发布 Kimi K3，并披露其具备 2.8T 总参数、100 万 token 上下文、原生多模态输入、Kimi Delta Attention 和 Attention Residuals 等核心特性，同时已接入 Kimi Work、Kimi Code 和 API，且承诺在 2026 年 7 月 27 日前开放权重。这一发布让 Kimi 不再停留在单点模型对比，而是以完整产品入口和开源节奏直接参与全球前沿模型竞争。

Arena 2026-07-17

Kimi K3 在 Frontend Code Arena 升至第一

Arena 报告显示，Kimi K3 以 1679 分升至 Frontend Code Arena 第一，在 7 个前端领域里拿下 6 项第一、游戏场景排名第二。这个结果的重要性在于，它把“开源模型接近前沿”推进到“在高可见度生产任务中直接领先”，也让前端生成、产品原型和视觉化编码成为 Kimi K3 最先形成冲击力的入口场景。

Moonshot AI / vLLM 2026-07-17

Moonshot 向 vLLM 贡献 KDA 前缀缓存支持

围绕 Kimi K3 的讨论迅速从能力扩展到系统实现：KDA 被解读为一种更高效的长上下文记忆机制，而 vLLM 则披露 Moonshot 已直接贡献 KDA 前缀缓存支持，为权重开放后的即时部署铺路。它的意义在于，前沿模型竞争不再只是参数和训练规模之争，架构创新是否能被开源推理栈快速吸收，正在决定模型真实可用性的上限。

OpenAI 2026-07-17

OpenAI 说明 Codex 文件删除事故的触发条件

OpenAI 工程负责人 Thomas Sottiaux 公开说明，少数 GPT-5.6 / Codex 文件删除事故主要出现在无沙箱、无自动审查的全权限环境中，且与错误处理临时目录和 `$HOME` 的行为有关。它的价值在于，这类案例把 Agent 风险从抽象安全讨论拉回到真实工作流，提醒行业必须把默认权限、隔离策略和执行护栏当成产品核心，而不是附属选项。

OpenAI Devs 在 Codex 中加入 PR Chat 和行内编辑

OpenAI Devs 宣布 Codex 新增 PR Chat 和 inline code editing，可在拉取请求上下文中直接审阅和修改代码，并同步为 GPT-5.6、ChatGPT 与 Codex 组织 Office Hours。它反映出 Codex 的重点正在从单纯执行代码任务，进一步延展到更完整的评审与协作工作流。

IT之家 / OpenAI 2026-07-13

OpenAI 暂时取消 Codex 和 ChatGPT Work 的 5 小时使用限制

根据 OpenAI Codex 工程负责人对外说明，Codex 和 ChatGPT Work 已暂时取消 Plus、Business 与 Pro 订阅的 5 小时使用限制，同时模型侧也在继续优化效率，以减少同等额度下的消耗。这个调整表明，前沿 Agent 产品在快速增长期仍在持续修正可用量、限额策略和成本体验之间的平衡。

Perplexity 2026-07-14

Perplexity 开源 WANDR Agent 研究评测集

Perplexity 推出 WANDR，这是一套基于脱敏生产研究任务构建的 500 题评测，包含 170,495 条带来源证据的记录，并通过重新抓取网页来校验答案而非依赖静态标准答案。它值得跟踪，因为 Agent 评测正在从“做出像样回答”转向“能否在变化中的真实互联网里完成可核查研究任务”，这会更直接影响深度研究产品与训练闭环的方向。

NVIDIA 2026-07-17

NVIDIA 发布 Nemotron 3 Embed 8B 及 1B 系列检索模型

NVIDIA 发布 Nemotron 3 Embed 8B，并强调其在 RTEB 等检索基准上的领先表现，同时提供 1B BF16 与 1B NVFP4 变体，强化在 Blackwell 平台上的吞吐效率。它的重要性在于，检索质量提升不只是向量数据库层面的更新，而是会直接减少下游 Agent 的冗余搜索与上下文 token 消耗，进而改写系统级成本结构。

Google 2026-07-17

Google 在 Google Vids 中接入 Gemini Omni 和个人 Avatar 生成功能

Google 宣布在 Google Vids 中接入 Gemini Omni，用于视频生成和编辑，并加入基于自拍与语音样本创建个人分身的能力，同时配套 SynthID 水印与账号级肖像限制。它值得关注，因为视频生成正在脱离独立创作工具，转而嵌入协作办公产品，AI 多模态能力正被直接包装成企业内容生产的默认能力层。

01

Kimi K3 在 Frontend Code Arena 升至第一

Arena 报告显示，Kimi K3 以 1679 分升至 Frontend Code Arena 第一，在 7 个前端领域里拿下 6 项第一、游戏场景排名第二。这个结果的重要性在于，它把“开源模型接近前沿”推进到“在高可见度生产任务中直接领先”，也让前端生成、产品原型和视觉化编码成为 Kimi K3 最先形成冲击力的入口场景。

02

Moonshot 向 vLLM 贡献 KDA 前缀缓存支持

围绕 Kimi K3 的讨论迅速从能力扩展到系统实现：KDA 被解读为一种更高效的长上下文记忆机制，而 vLLM 则披露 Moonshot 已直接贡献 KDA 前缀缓存支持，为权重开放后的即时部署铺路。它的意义在于，前沿模型竞争不再只是参数和训练规模之争，架构创新是否能被开源推理栈快速吸收，正在决定模型真实可用性的上限。

03

OpenAI 说明 Codex 文件删除事故的触发条件

OpenAI 工程负责人 Thomas Sottiaux 公开说明，少数 GPT-5.6 / Codex 文件删除事故主要出现在无沙箱、无自动审查的全权限环境中，且与错误处理临时目录和 `$HOME` 的行为有关。它的价值在于，这类案例把 Agent 风险从抽象安全讨论拉回到真实工作流，提醒行业必须把默认权限、隔离策略和执行护栏当成产品核心，而不是附属选项。

04

OpenAI Devs 在 Codex 中加入 PR Chat 和行内编辑

OpenAI Devs 宣布 Codex 新增 PR Chat 和 inline code editing，可在拉取请求上下文中直接审阅和修改代码，并同步为 GPT-5.6、ChatGPT 与 Codex 组织 Office Hours。它反映出 Codex 的重点正在从单纯执行代码任务，进一步延展到更完整的评审与协作工作流。

05

OpenAI 暂时取消 Codex 和 ChatGPT Work 的 5 小时使用限制

根据 OpenAI Codex 工程负责人对外说明，Codex 和 ChatGPT Work 已暂时取消 Plus、Business 与 Pro 订阅的 5 小时使用限制，同时模型侧也在继续优化效率，以减少同等额度下的消耗。这个调整表明，前沿 Agent 产品在快速增长期仍在持续修正可用量、限额策略和成本体验之间的平衡。

06

Perplexity 开源 WANDR Agent 研究评测集

Perplexity 推出 WANDR，这是一套基于脱敏生产研究任务构建的 500 题评测，包含 170,495 条带来源证据的记录，并通过重新抓取网页来校验答案而非依赖静态标准答案。它值得跟踪，因为 Agent 评测正在从“做出像样回答”转向“能否在变化中的真实互联网里完成可核查研究任务”，这会更直接影响深度研究产品与训练闭环的方向。

07

NVIDIA 发布 Nemotron 3 Embed 8B 及 1B 系列检索模型

NVIDIA 发布 Nemotron 3 Embed 8B，并强调其在 RTEB 等检索基准上的领先表现，同时提供 1B BF16 与 1B NVFP4 变体，强化在 Blackwell 平台上的吞吐效率。它的重要性在于，检索质量提升不只是向量数据库层面的更新，而是会直接减少下游 Agent 的冗余搜索与上下文 token 消耗，进而改写系统级成本结构。

08

Google 在 Google Vids 中接入 Gemini Omni 和个人 Avatar 生成功能

Google 宣布在 Google Vids 中接入 Gemini Omni，用于视频生成和编辑，并加入基于自拍与语音样本创建个人分身的能力，同时配套 SynthID 水印与账号级肖像限制。它值得关注，因为视频生成正在脱离独立创作工具，转而嵌入协作办公产品，AI 多模态能力正被直接包装成企业内容生产的默认能力层。
